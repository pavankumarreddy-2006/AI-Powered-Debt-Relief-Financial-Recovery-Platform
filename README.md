# ANTI DEBT: AI-Powered Debt Relief & Financial Recovery Platform

This repository is structured according to the **AI-ML and Generative AI Track Project Template** layout guidelines.

## 📂 Repository Structure

The project deliverables are organized into the following phases:

1. **[1. Brainstorming & Ideation](./1.%20Brainstorming%20%26%20Ideation/)**: Idea prioritization, problem statement scoping, empathy mapping, and user maps.
2. **[2. Requirement Analysis](./2.%20Requirement%20Analysis/)**: Detailed requirements matrices, technology stack parameters, and data flow diagrams.
3. **[3. Project Design Phase](./3.%20Project%20Design%20Phase/)**: Solution architecture blueprinting and proposed mock designs.
4. **[4. Project Planning Phase](./4.%20Project%20Planning%20Phase/)**: Development milestones, schedules, and task boards.
5. **[5. Project Development Phase](./5.%20Project%20Development%20Phase/)**: **Contains the active codebase files (FastAPI Backend and Vite React Frontend)**.
   * Go directly to the **[Codebase Directory](./5.%20Project%20Development%20Phase/)** to run the local developer servers or containerized docker-compose builds.
6. **[6. Project Testing](./6.Project%20Testing/)**: Walkthrough reports, automated test coverages, and performance benchmarks.
7. **[7. Project Documentation](./7.Project%20Documentation/)**: User manuals, system guides, and execution specifications.
8. **[8. Project Demonstration](./8.Project%20Demonstration/)**: Features demonstration guides, planning checklists, and future scalability architectures.
